# Helpdesk (Tickets)

Support ticketing ported from the upstream RMS codebase into Holt's idiom (Int IDs,
`organizationId` multi-tenancy, `created/updated/createdBy/updatedBy` audit
fields). Gated by the `helpdesk` feature flag (`lib/featureCatalog.ts`,
default **on**). When the flag is off the nav item hides, the staff pages
redirect home, and the public `/support` pages return 404.

## Data model (`prisma/schema.prisma`)

- **Ticket** — one support request.
  - `ticketNumber` (unique, `TKT-YYMMDD-NNN`) — human handle.
  - `publicToken` (unique, 192-bit random) — stable handle for the no-login
    status view. Generated by `lib/tickets/token.ts`.
  - Origin is either a known `customer` (FK) **or** an anonymous public
    submitter (`submitterName` + `submitterEmail`). Both nullable.
  - `status` (`TicketStatus`), `priority` (`TicketPriority`), optional
    `assignedTo` (FK → `StaffMember`, relation `TicketAssignee`).
  - `resolvedAt` — stamped when status enters RESOLVED/CLOSED, cleared on reopen.
- **TicketMessage** — a thread entry.
  - Authored by a staff member (`authorStaff` FK, relation
    `TicketMessageAuthor`) or by the customer (`authorName`).
  - `isInternal` — staff-only note; **never** returned by the public endpoint.

### Enums

- `TicketStatus`: `OPEN`, `IN_PROGRESS`, `WAITING_ON_CUSTOMER`, `RESOLVED`, `CLOSED`.
- `TicketPriority`: `LOW`, `MEDIUM`, `HIGH`, `URGENT`.

Migration: `prisma/migrations/20260603212811_add_helpdesk_tickets`.

## Contract + helpers (`lib/tickets/`)

Shared client/server contract (CLAUDE.md rule 7) lives in
`ticketContract.ts` — both the staff UI and the server validators import the
value lists, labels, and predicates from one place:

- `TICKET_STATUS_VALUES` / `TICKET_PRIORITY_VALUES` (+ `*_LABELS`).
- `isValidTicketTransition(from, to)` / `getValidTicketTransitions(from)` — the
  allowed status graph. Forward through the lifecycle, bounce to the customer,
  reopen from a done state. A same-status save is a valid no-op.
- `isOpenTicketStatus` (OPEN/IN_PROGRESS/WAITING = queue) /
  `isResolvedTicketStatus` (RESOLVED/CLOSED = stamps `resolvedAt`).
- `TICKET_PRIORITY_RANK` — URGENT→0…LOW→3 for queue sorting.

Other helpers:

- `numbering.ts` — `ticketNumberPrefix(date)` + `nextTicketNumber(prefix, last)`
  (pure, tested) and `generateTicketNumber()` (DB), house style mirroring
  `lib/returnService.ts`.
- `token.ts` — `generateTicketToken()` (`randomBytes(24).base64url`).
- `requestBody.ts` — zod parsers: `parseTicketCreateInput` (public submit),
  `parseTicketUpdateInput` (staff triage, ≥1 field), `parseTicketMessageInput`.

Tests: `__tests__/ticketContract.test.ts`, `ticketNumbering.test.ts`,
`ticketRequestBody.test.ts`.

## API (`pages/api/tickets/`)

| Route | Method | Auth | Purpose |
|---|---|---|---|
| `/api/tickets` | GET | SUPER_ADMIN/ADMIN/MANAGER | Queue list; `?status=open\|<status>`, `?assignedToId=<id>\|unassigned`, `?q=` search |
| `/api/tickets` | POST | public (rate-limited 5/min) | Open a ticket; first message stored as a non-internal `TicketMessage`; returns `{ ticketNumber, publicToken }` |
| `/api/tickets/[id]` | GET | staff | Full ticket incl. internal notes |
| `/api/tickets/[id]` | PATCH | staff | Triage: status (transition-validated) / priority / assignee / subject; manages `resolvedAt` |
| `/api/tickets/[id]/messages` | POST | staff | Add a public reply or internal note; author resolved from session → StaffMember |
| `/api/tickets/public/[token]` | GET | public | Status + public thread (internal notes filtered) |
| `/api/tickets/public/[token]` | POST | public (rate-limited) | Customer reply; reopens a WAITING/RESOLVED ticket to OPEN |

Handlers stay thin (CLAUDE.md rule 14): validation lives in `requestBody.ts`,
transition rules in `ticketContract.ts`. Search uses `buildSearchFilter`
combined as `{ AND: [filters, searchFilter] }`.

## UI

- Staff queue: `app/(dashboard)/app/helpdesk/page.tsx` + `HelpdeskQueueView.tsx`
  — Open/All filter, search, open-first/priority/recency sort, status+priority
  badges, links to the detail workspace.
- Staff detail: `app/(dashboard)/app/helpdesk/[id]/` — triage controls
  (status limited to valid transitions), threaded messages (internal notes
  styled amber), reply box with internal-note toggle, "copy customer link".
- Public submit: `app/(site)/support/page.tsx` + `SupportFormView.tsx` —
  on success shows the ticket number + a "track your request" link.
- Public status: `app/(site)/support/[token]/` — status + thread + reply;
  `noindex`.

Nav: `Helpdesk` in `lib/auth/navPermissions.ts` (`NAV_ITEMS`,
`NAV_FEATURE_KEYS.Helpdesk = "helpdesk"`, default roles SUPER_ADMIN/ADMIN/MANAGER).

## Verification checklist

- [ ] `npm run validate` (lint + typecheck + format) clean.
- [ ] Ticket lib unit tests pass.
- [ ] Public submit creates a ticket + first message; returns a token link.
- [ ] Internal notes never appear on `/api/tickets/public/[token]`.
- [ ] Status PATCH rejects an illegal transition (e.g. CLOSED → IN_PROGRESS).
- [ ] Customer reply reopens a RESOLVED/WAITING ticket.
- [ ] With `helpdesk` off: nav hidden, `/app/helpdesk` redirects, `/support` 404s.

## Not ported (deliberate)

The upstream RMS also modelled `File` attachments on tickets/messages. Not ported in
this slice — Holt's upload path (`lib/secureUpload.ts`) can be wired to
`TicketMessage` later if needed. Tracked as a follow-up, not a gap.
